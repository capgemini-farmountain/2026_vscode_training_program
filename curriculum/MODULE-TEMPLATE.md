# Module Template Reference

Use this checklist when writing any new module. Every module must have exactly four files.

---

## README.md

- [ ] Header: path, session number, estimated time, NotebookLM link
- [ ] Numbered learning objectives (Feynman-testable — can you explain each simply?)
- [ ] One theory section per objective — analogy first, precision second
- [ ] Tables for any comparisons
- [ ] SDD principle referenced at least once
- [ ] Feynman test at the end (one plain-language explanation challenge)
- [ ] "Next" link pointing to `lab.md`

## lab.md

- [ ] Time estimate at top
- [ ] "Before you start" setup section
- [ ] Steps with checkboxes `- [ ]`
- [ ] Recording template (markdown table or structured format) for each experiment
- [ ] "Submit" section with Google Classroom instructions
- [ ] Lab produces a named artifact the learner keeps

## quiz.md

- [ ] Exactly 5 questions
- [ ] All multiple choice
- [ ] Correct answer marked with `✓`
- [ ] One-line explanation per correct answer
- [ ] Questions test the module learning objectives, not trivia

## reflection.md

- [ ] Exactly 3 questions
- [ ] Each question asks the learner to reference their lab data
- [ ] Written in second person ("Describe a task you do...")
- [ ] No yes/no questions — all require explanation

---

## Content rules (apply to all files)

- SDD-only: never present vibe coding as a valid professional approach
- No TBD, TODO, or placeholder content in published modules
- Every lab ends with an artifact committed to the learner's Superpowers fork (M2 onwards)
- Feynman test is mandatory in every README.md
