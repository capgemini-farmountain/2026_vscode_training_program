# Contributing

## Adding a module

Each module lives in `curriculum/mX-name/` or `curriculum/cX-name/` and contains:

- `README.md` — module theory (this is the NotebookLM source)
- `lab.md` — step-by-step lab instructions
- `quiz.md` — 5 questions for Google Classroom import
- `reflection.md` — 3 reflection prompts for submission

## Adding a plugin

See the [plugin marketplace](https://github.com/capgemini-farmountain/copilot-plugins) repo.

## Style guide

- SDD-only: never present vibe coding as a valid professional approach
- Every module ends with a Feynman test (plain-language explanation challenge)
- Every lab produces an artifact the learner keeps
- No TBD, TODO, or placeholder content in published modules
